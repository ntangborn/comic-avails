## ⭐ The two items to see real data

The app is fully built and running, but shows empty states because (a) no
solicitations have been ingested and (b) the pull list isn't wired to a user.
Both come down to a few `.env.local` edits and one command.

### Item 1 — Load catalog data (makes `/` and `/week` show real books)

1. **Fix two keys in `.env.local`** (both currently hold the publishable key by
   mistake):
   - `ANTHROPIC_API_KEY=sk-ant-...` — from console.anthropic.com → API Keys
   - `SUPABASE_SECRET_KEY=sb_secret_...` — Supabase → Settings → API Keys →
     Secret keys → Create secret key (name it `ingest-scripts`)
2. **Run the DC ingest:**
   ```
   npx tsx scripts/ingest-dc.ts https://www.comicreleases.com/2026/06/dc-september-2026-solicitations/
   ```
   - Healthy run: ~150–300 extracted, 90%+ upserted, a few sensible rejects.
   - **Watch for a 403** — comicreleases.com is behind Cloudflare and blocked our
     earlier fetch. If the script 403s, tell me and I'll add a browser-like
     User-Agent / fallback. (This is the one unproven risk.)
   - If it errors `Publisher 'dc' not found`, the migration isn't applied — paste
     `supabase/migrations/20260703120000_init_comic_avails_schema.sql` into the
     Supabase SQL Editor and run it, then retry. *(It appears already applied —
     the catalog query succeeds against an empty `items` table with no error — but
     confirm if you see that message.)*
3. **Trust-check** (build guide §3.4): Supabase → Table Editor → items; hand-check
   ~10 rows against the solicit page. Tell me any systematic pattern to fix (e.g.
   "omnibuses tagged trade_paperback") — re-running is safe (idempotent upsert).

Refresh `http://localhost:3000` and the FOC view fills in.

### Item 2 — Enable the pull list (makes "+ Add", qty, remove persist)

`pull_lists.user_id` is a foreign key to Supabase Auth, so the single hardcoded
user needs to be a **real** auth user.

1. **Create one user:** Supabase → Authentication → **Add user** (any email /
   password). Copy its **User UID** (a UUID).
2. **Add to `.env.local`:**
   ```
   DEMO_USER_ID=<paste the UUID>
   ```
   (also needs `SUPABASE_SECRET_KEY` from Item 1; optional:
   `PULL_LIST_SHOP_NAME`, `PULL_LIST_CUSTOMER_NAME`)
3. **Restart the dev server** so it picks up the new env (`npm run dev`).

Until this is done, `/pull-list` correctly shows "not configured yet" and Add
buttons report the same — that's expected, not a bug.

> Env changes require a dev-server restart. There's currently a dev server on
> port 3000 (PID 57220) serving the latest code; stop it with
> `taskkill /PID 57220 /F` (or just start fresh after editing env).

## What was built this session (Part 4.1)

`AGENTS.md`) — this Next.js makes `params`/`searchParams` Promises and exports
`revalidatePath` from `next/cache`.

- **`/`** — "This Week's FOC": items with `foc_date` in the next 7 days, grouped
  by FOC date → publisher, color-coded countdown badges (red urgent / amber soon).

  variants.

- Variant collapse ("+N variants" expander), filters (publisher multi-select,

**New files:** `app/{page,layout}.tsx`, `app/week/**`, `app/item/[id]/page.tsx`,
`app/pull-list/**`, `components/*` (12), `lib/{catalog,pull-list,format,types}.ts`,

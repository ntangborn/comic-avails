## B. Tooling / environment health

### B1. Claude Code "auto update failed" (claude.exe / claude.ps1)  🔧
Cause: Claude Code is installed via **npm-global**
(`@anthropic-ai/claude-code@2.1.201` at `C:\Users\ntang\AppData\Roaming\npm\`).
On Windows, npm-global auto-updates commonly fail on directory permissions —
that's this error. `claude migrate-installer` is **not** a real command; the fix
is to switch to the native installer, which self-updates without permission
issues.

Run these **in a normal PowerShell window, not inside a Claude Code session**
(the uninstall would kill a running session):
```powershell
claude doctor          # diagnose install health + why auto-update failed
npm uninstall -g @anthropic-ai/claude-code
irm https://claude.ai/install.ps1 | iex   # native installer -> %USERPROFILE%\.local\bin\claude.exe
claude --version       # confirm (reopen the terminal first so PATH refreshes)
```
Native installer: no admin needed, PATH handled automatically, background
updates. Manual-update fallback on any install: `claude update`.

### B2. npm audit vulnerabilities  🔧
`npm install` reported vulnerabilities (it suggested `npm audit fix --force`).
Review before acting — `--force` pulls breaking major-version changes and could
break `next`/`tsx`/`supabase-js`.
```powershell
npm audit            # read the report first
npm audit fix        # safe, non-breaking fixes only (skip --force for now)
```
Low urgency for a personal tool, but worth a look before deploy (Part 5).

### B3. Supabase CLI not installed  🔧 (optional)
Not required — the dashboard SQL Editor works for A2. Install only if you prefer
`supabase db push` / migration tracking:
```powershell
npm i -g supabase    # or scoop install supabase
```

### B4. Vercel CLI — already installed ✅ (informational)
